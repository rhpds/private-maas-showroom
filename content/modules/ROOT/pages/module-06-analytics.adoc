= Usage Analytics and Reporting

In this module, you're stepping into the role of a technical decision maker or platform stakeholder - someone responsible not just for building AI services, but for understanding and reporting on their business impact.

You've deployed, exposed, and consumed a model. Now it's time to ask:

* How is it being used?
* Who's using it the most?
* What's the cost and what's the return?

Let's walk through how to leverage a Grafana dashboard and platform level metrics from the Prometheus observability stack to answer those questions.

NOTE: For this workshop, we've created a **custom Grafana dashboard** that integrates with Red Hat AI's built-in observability data sources, specifically the Prometheus metrics stack included with the product. While this particular dashboard is a custom workshop example and not a product feature, it demonstrates how you can leverage the platform's existing Prometheus metrics to build usage analytics and reporting. Red Hat AI will include integrated analytics capabilities in upcoming releases.

== Why This Matters

As AI adoption grows, visibility is critical. Organizations need to track: 

* Model/API usage by team or model
* Call volume over time
* Popular endpoints
* Cost-to-value
* Infrastructure impact (e.g. GPU usage)

This insight enables:

* Cost attribution (per team, per model, per environment)

* Capacity planning (predict future infrastructure scaling needs)

* Understanding ROI for AI workloads

* **Model** - Only one model is deployed in this workshop

=== Understanding the Dashboard Metrics

**Total Authorized Hits**

This represents the total token count (both input and output tokens combined) for all authorized requests to the model.

**Authorized Hits by User and Tier**

Breaks down token usage by individual user and their assigned tier. For this workshop, all users are assigned to the Enterprise tier.

**Top 10 Users by Hits**

Displays the top users ranked by total token usage. Look for your username to see where you rank compared to other workshop participants.

**Hourly Authorized Hits by User**

Shows token usage over time in hourly increments, filtered by user. This helps identify usage patterns and peak activity periods.

**Total Authorized Hits by Tier (Time Range)**

Aggregates token usage by tier within the selected time range. Useful for understanding tier-level consumption.

**Top 5 Users by Cost (Based on Tier Pricing)**

Ranks users by estimated cost based on tier pricing models. In enterprise environments, this metric drives chargeback and cost attribution decisions.

**Detailed Metrics Table**

Provides granular per-user metrics including total tokens, costs, and activity timestamps. Use this table to audit specific user activity or investigate usage anomalies.

== Making Data-Driven Decisions

This dashboard enables critical business and operational decisions:

**Capacity Planning**: If token usage doubles month-over-month, you need proportional GPU scaling. Identifying growth trends early prevents performance degradation and justifies infrastructure investment.

**Cost Attribution**: The "Top 5 Users by Cost" metric enables chargeback models. If one team consumes 80% of tokens, that data justifies dedicated capacity allocation and budget decisions.

**ROI Demonstration**: Correlate token usage with business outcomes. High adoption rates combined with faster development cycles (from earlier modules) prove clear return on investment - use this to justify expansion.

**Optimization**: Hourly usage spikes might indicate batch processing, inefficient prompting, or peak adoption periods. This insight drives optimization strategies and resource scheduling.
