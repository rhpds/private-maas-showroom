= System Administration with Agentic AI 

In the last module, you stepped into the developer role building applications using private LLM endpoints. Now, you'll become a **site reliability engineer (SRE) or DevOps practitioner** focused on observability, diagnostics, and operational scale. 

But you won't go at it alone, you'll work with agentic AI tooling to navigate your OpenShift environment in real time - just like a modern SRE team building insight-driven automation.

== Why Agentic AI for Admins?

Think of agentic AI as your copilot for cluster operations - ready to answer complex infrastructure questions using reasoning and real-time data.

Instead of navigating complex dashboards, YAML, and shell scripts, you'll try something new:

* “What’s consuming the most memory in our dev cluster right now?”

* “Were there any pod crashes in the last 30 minutes?”

* “Why did that Job fail this morning?”

These natural-language prompts get translated into real queries and results. The outcome? Faster issue resolution, increased system transparency, and lower barrier to operational insights for both technical and non-technical users.

That's an example of the power of agentic AI.

== Meet Your New Toolkit

In this module, we'll use a preconfigured stack that includes:

* **Gen AI Studio Playground** - an integrated UI feature in Red Hat AI for testing model responses, MCP server integrations, and tool invocations before committing to production code. Use it to validate that models, prompts, and external connections work together correctly, then export the validated configuration to your application.

* **Kubernetes MCP Server**  - allows our LLM agent to interact with our live OpenShift cluster and deployed resources.

* **Rocket.Chat MCP Server** - allows our LLM agent to interact with a Rocket.Chat workspace. We will use it to interact with a specific Rocket.Chat workspace for which we have authorization configured.

* **NVIDIA Nemotron 3 Nano 30B A3B FP8 Model** - a lightweight, instruction-tuned model optimized for agentic workflows and tool calling. The FP8 quantization enables efficient inference while maintaining accuracy for structured tasks like API calls and multi-step reasoning. This model drives the agent's decision-making as it translates natural language into MCP server tool invocations.

=== Using the playground

This is where the magic happens. The playground lets you test-drive your entire agentic AI stack - model, prompts, MCP servers, and tool calls - **before writing a single line of application code**.

Think of it as your **sandbox for operational automation**. Instead of deploying blind and debugging in production, you validate the whole chain here:

* Does the model understand my prompts correctly?
* Are the MCP servers responding with the right data?
* Can I get pod logs, list namespaces, or post to chat channels reliably?

**The workflow:** Experiment here → validate behavior → export configuration → integrate into automation.

In the playground, you'll see a chat interface with your model preselected and a default system prompt. You can adjust the system prompt or temperature as needed.

NOTE: The **temperature** setting controls model creativity. Keep it **low** (0.0-0.3) for deterministic, accurate responses - critical when invoking tools or querying infrastructure. Higher temperatures introduce randomness, which can break tool calls.

image::playground.png[]

== Investigate our OpenShift Resources

The tool information will give you guidance into how to interact with the model in chat to activate the tool calls correctly. You will need to utilize the right keywords so that you're engaging with the appropriate tools.

In the chat, enter:

[source,console,role=execute,subs=attributes+]
----
List all pods in the grafana namespace.
----

Response output will vary slightly, but you should see it correctly list the active pod in the `grafana` namespace: `grafana-deployment`.

Let's try something else:

[source,console,role=execute,subs=attributes+]
----
Get logs for the grafana-deployment pod in the grafana namespace.
----

Feel free to experiment further with the tools available.

=== Post a Message to our Rocket.Chat Workspace

With our Rocket.Chat MCP Server, we can extend our agentic AI experience beyond Kubernetes and into team collaboration tools (among many other possibilities!).

This MCP server bridges your NVIDIA Nemotron model with a Rocket.Chat workspace to fetch and post approved data.

Open the Rocket.Chat application at: {rocketchat_url}

Login with the following credentials:

* **Username**: {user}
* **Password**: {password}

You will see a pop-up. Simply close it, or accept update configuration:

image::rocketchat-popup.png[]

NOTE: The Rocket.Chat workspace is pre-configured with a `#workshop` channel where your AI agents will post messages. You will observe the messages there after sending them through the playground.

**Why this matters:**

* SREs and DevOps teams often work across multiple collaboration channels.

* By giving your AI visibility into a collaboration space like Rocket.Chat (similar to Slack), you can use natural language to check team communication spaces or to post information without switching tools.

* This is a safe, controlled example — you can read channels and post messages through your authorized bot user.

==== In the Gen AI Studio Playground chat interface, type:

[source,console,role=execute,subs=attributes+]
----
List all Rocket.Chat channels in our workspace.
----

Now, let's post a message to our Rocket.Chat workspace:

[source,console,role=execute,subs=attributes+]
----
Post a message to the #workshop channel: "Hi from <insert name>".
----

Go check the `#workshop` channel in the Rocket.Chat application to see your message.

==== Send logs to Rocket.Chat

Now, let's try out a real use case for this! It may not be done through a chat UI like the playground, but it's a good example of how you can use agentic AI to help you with your work. In a production environment, you would likely use a more robust automation to send logs or other information from your Red Hat AI cluster to Rocket.Chat.

Let's send pod logs from your workspace to the `#workshop` channel, where you and your team can view them in the Rocket.Chat interface.

[source,console,role=execute,subs=attributes+]
----
Get the pod logs for the grafana-deployment pod in the grafana namespace. Assess the logs and post the logs + summary message to the #workshop channel in Rocket.Chat.
----
