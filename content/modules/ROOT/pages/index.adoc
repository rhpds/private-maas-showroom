= Private Model as a Service - A Practical Introduction with Red Hat AI

== Welcome to the Workshop

In this **lightning lab**, you'll experience Models-as-a-Service in action. When AI infrastructure lacks governance, teams duplicate models, GPUs sit idle or overload, and costs escalate without accountability.

**Models-as-a-Service** solves this by centralizing model deployment and exposing them through secure, authenticated APIs. Platform teams manage models once; developers consume them without GPU complexity.

You'll experience three key aspects:

* **Model Access**: Retrieve credentials from a centrally-deployed NVIDIA Nemotron model through the Red Hat OpenShift AI dashboard
* **Agentic AI**: Use natural language to interact with Rocket.Chat via the Gen AI Studio Playground
* **Usage Analytics**: View real-time token consumption and cost metrics through a custom Grafana dashboard

By the end, you'll see how MaaS eliminates GPU waste, enables cost attribution, and transforms AI infrastructure into a managed service.

== Workshop Environment

You'll work in a shared Red Hat AI cluster environment with your own assigned user access. All participants share access to a centrally-deployed **NVIDIA Nemotron 3 Nano 30B A3B FP8** model.

This reflects a real-world Models-as-a-Service architecture: a platform team deploys and manages the model infrastructure, while users consume it via standard API endpoints. The model runs on GPU-accelerated OpenShift nodes and serves requests through Red Hat AI's inference stack.

**Why NVIDIA Nemotron for this lab?**

* **Enterprise-optimized**: FP8 quantization delivers high throughput on modern GPUs
* **Versatile**: Handles reasoning and tool-calling via MCP servers
* **Production-ready**: Instruction-tuned for reliable, deterministic responses
* **Cost-effective**: 30B parameters provide strong capability without excessive resource requirements

You'll use this same model across all modules - from accessing credentials to agentic automation to viewing usage analytics.
