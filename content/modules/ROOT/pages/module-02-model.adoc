= Introduction to your environment

Our first step is to get accustomed to our environment and its components.

In this workshop, you'll work with **NVIDIA Nemotron 3 Nano 30B A3B FP8** - a lightweight, instruction-tuned large language model designed for enterprise deployment. With FP8 quantization, it delivers production-grade inference performance while maintaining accuracy for coding assistance, reasoning tasks, and tool-calling workflows. This makes it ideal for Models-as-a-Service architectures where multiple teams share centrally-managed model infrastructure.

== View model deployment

OpenShift AI provides a web-based dashboard interface to deploy and manage models with ease, at scale and in conjunction with other critical AI assets like MCP servers or evaluation tooling.

Let's get connected.

**Open the OpenShift AI Dashboard**

Click the navigation button and select **Red Hat OpenShift AI** from the drop-down menu.

image::nav-rhoai.png[]

If prompted, login again with your user credentials to the OpenShift AI dashboard.

Once authenticated you will see the following home page:

image::rhoai-dashboard.png[]

Now, go to the `Gen AI Studio` section of the dashboard and select `AI asset endpoints`.

image::nav-genaistudio.png[]

In the AI asset endpoints page, change the project from `grafana` to {user}.

image::genaistudio-changeproject.png[]

Now you may view your available asset endpoints.

We currently have a model deployed in a namespace managed by our cluster admin. This admin has set up a `Models as a Service` architecture for our organization with a central model they have deployed and managed that is available for use by remote teams across the organization.

We do not have access to manage the model deployment directly, but we can access its endpoint information via the `Models as a Service` tab. 

image::maas-tab.png[]

Select the tab. Your admin has deployed one NVIDIA model for you to use: `nemotron-3-nano-30b-a3b`

image::maas-model.png[]

Let's grab the endpoint to use in our code assistant. Select `View` under the `External endpoint` column:

image::view-endpoint.png[]

Copy the MaaS route and save it in a note file - you'll need this in the next module.

NOTE: To copy the route, highlight the text and use kbd:[Ctrl+C] (or kbd:[Cmd+C] on Mac).

image::copy-route.png[]

Now, click `Generate API token`.

image::generate-token.png[]

Copy this API token and save it with your route endpoint - you'll need both values in the next module.

NOTE: Highlight the token text and copy with kbd:[Ctrl+C] (or kbd:[Cmd+C] on Mac).

image::copy-token.png[]

Now that we have our model connection details, we can proceed to our development environment to leverage the model.
