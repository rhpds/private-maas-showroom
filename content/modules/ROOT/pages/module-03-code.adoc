== Setting up a cloud integrated developer environment (IDE) with OpenShift Dev Spaces

OpenShift Dev Spaces delivers instant access to pre-configured, containerized workspaces running on your OpenShift cluster. It ensures consistency across your team, eliminates "works on my machine" problems, and keeps your development environment in sync with your deployment target.

**Why use Dev Spaces for this workshop?** While you can expose model endpoints externally and develop locally, Dev Spaces offers advantages: your code runs in the same environment where it will deploy, dependencies are pre-configured, and you can access cluster-internal services directly. For this workshop, the model is cluster-internal only, making Dev Spaces the natural choice.

NOTE: The OpenShift AI dashboard shows "External endpoint" for the route — this is currently the default MaaS labeling in the UI. In this environment, the route is cluster-internal only.

== Meet Your AI Code Assistant

**Continue** is an open-source AI code assistant that integrates into VS Code. Continue provides an interactive chat interface where you can have natural conversations about your code. You can ask Continue to perform actions like:

**“Add logging to this function”**

**“Generate a unit test for this file”**

**“Refactor this into smaller components”**

**“Explain what this code does”**

Continue will execute these requests directly in your editor, with full awareness of your codebase and project context.

Continue supports custom model endpoints, making it a perfect example for connecting to your private enterprise models. Continue is an **open-source** solution that gives you complete control over your AI coding workflow. 

NOTE: Continue is an example of an AI-powered coding assistant for this workshop experience. You can easily imagine how this could work with any other desired code assistant extensions that allow you to customize the model used.

=== Configure the Continue Extension

We've provided a pre-configured template file for the Continue extension. You'll need to move it to the proper location where Continue expects to find it.

Open a terminal in VS Code (Terminal > New Terminal from the menu, or kbd:[Ctrl+`])

image::new-terminal.png[]

In the terminal, run the following command:

[source,sh,role="execute"]
----
cp /projects/exercises/.vscode/config.yaml ~/.continue/config.yaml
----

This copies the template configuration file to `~/.continue/config.yaml`, replacing the blank template that's there.

Now open the configuration file to customize it with your MaaS endpoint details. Open the search bar (kbd:[Cmd+P] on Mac or kbd:[Ctrl+P] on Linux) and type:

[source,sh,role="execute"]
----
~/.continue/config.yaml
----

image::search-config.png[]

Hit `Enter` to open the file. You'll see the following configuration:

[source,yaml,role="execute"]
----
name: Local Assistant
version: 1.0.0
schema: v1
models:
  - name: NVIDIA Nemotron 3 Nano 30B A3B FP8
    provider: openai
    model: "nemotron-3-nano-30b-a3b"
    apiBase: "YOUR_MAAS_ROUTE/v1"
    apiKey: "YOUR_API_KEY"
context:
  - provider: code
  - provider: docs
  - provider: diff
  - provider: terminal
  - provider: problems
  - provider: folder
  - provider: codebase
----

IMPORTANT: Replace the `apiBase` URL with your actual MaaS endpoint URL and `YOUR_API_KEY` with the API key from your MaaS application. Ensure you retain the `v1` at the end of the path.

For example:

* **apiBase**: `http://maas.apps.ocp.75d7w.sandbox114.opentlc.com/llm/nemotron-3-nano-30b-a3b/v1` (Note the `v1` at the end of the URL)
* **apiKey**: `your-actual-api-key-here`

NOTE: You may reference the complete configuration documentation here: https://docs.continue.dev/reference[Continue Documentation^]

When the model is properly configured, you will be able to chat with the model in the sidebar.

image::ready-to-chat.png[]

Go ahead - test it out with the following prompt:

[source,sh,role="execute"]
----
Explain the software company Red Hat in a short paragraph.
----

NOTE: If you get an error when chatting with the model, double check your API key and route details. Ensure you have left the `/v1` at the end of the URL string.
